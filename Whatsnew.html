    Редактирование аранжировки
  • Аудиоклипы деформированной аранжировки теперь можно растянуть, удерживая нажатой клавишу Shift при перетаскивании их границы.
  • Теперь можно перемещать клипы аранжировки в пределах выделенного времени с помощью клавиш со стрелками влево / вправо. Чтобы переключить привязку к сетке, нажмите и удерживайте клавишу-модификатор «CMD» (Mac) / «Alt» (ПК).
  • Теперь можно перемещать содержимое клипа аранжировки внутри границ клипа, удерживая «Alt + Shift» (Mac) / «Ctrl + Shift» (ПК) и перетаскивая область отображения дорожки клипа. Чтобы переключить привязку к сетке, нажмите и удерживайте клавишу-модификатор «CMD» (Mac) / «Alt» (ПК).
  • В аранжировке команду «Реверс» теперь можно выполнить для выбора времени с помощью новой клавиши быстрого доступа «R». Также исправлена ​​ошибка, связанная с реверсированием семплов.
  • При использовании реверсирования для частично выбранного клипа в аранжировке реверсируется только выбранная часть.
  • Теперь можно деактивировать, обратить вспять и обрезать выбранный материал в течение заданного времени на полосе автоматизации.
  • Команда «Активировать / Деактивировать клип (ы)» теперь только (де) активирует выбранную часть клипов аранжировки. Раньше был (де) активирован весь клип, даже если была выбрана только его часть.
  • Клипы аранжировки теперь можно перемещать, перетаскивая верхнюю половину области содержимого клипа. Курсор мыши изменится на руку, чтобы указать перетаскиваемую область клипа, где время также можно выбрать одним щелчком мыши.
  • Выбор цикла в меню «Правка» теперь можно переключать, нажимая «CMD + L» (Mac) / «CTRL + L». (ПК). Раньше ярлык мог только включать цикл аранжировки, но не отключать его.
  • Теперь можно включить отслеживание нот MIDI с помощью команды «Chase MIDI Notes» в меню «Параметры».
  • Можно импортировать аудио- или MIDI-файл через меню «Создать». В зависимости от типа трека эта команда называется «Импортировать MIDI-файл…» или «Импортировать аудиофайл». Файл будет вставлен либо в позицию маркера вставки (вид аранжировки), либо в текущий выбранный пустой слот клипа (просмотр сеанса).
  • Двойной щелчок по MIDI-дорожке в режиме аранжировки теперь создает новый MIDI-клип.
  • Назначения MIDI и клавиш теперь можно назначить переключателю Lock Envelopes.
    Автоматизация
  • Реализован режим глобальной автоматизации. В расположении видимость всех дорожек автоматизации теперь переключается с помощью кнопки «Режим автоматизации» или клавиши «A». Когда полосы автоматизации видны, полосы контента минимизируются до высоты заголовков клипов.
  • Фейды были удалены из выбора Фейдов / Устройств в Аранжировке. Фейды клипа теперь всегда доступны на полосе контента, если дорожка имеет высоту не менее трех единиц.
  • Теперь можно разделить и объединить материал во временной выборке, даже если выбор существует в полосе автоматизации.
  • Теперь можно создавать точки останова автоматизации, дважды щелкнув в любом месте окна просмотра конверта / полосы автоматизации. В аранжировке это означает, что MIDI-клипы могут быть созданы только двойным щелчком на полосе содержимого.
  • При перемещении нескольких сегментов в пределах выделения времени привязка выравнивается по левому или правому краю выделения времени, в зависимости от того, с какой стороны выделения времени вы начинаете перетаскивать. Аналогичная логика применяется и к отдельным сегментам. Кроме того, при наведении курсора на сегменты теперь отображается значение выровненных событий. Значение автоматизации на границе выделенного времени также отображается, независимо от того, существует ли точка останова, при наведении курсора рядом с сегментом.
  • При создании точки останова автоматизации в выборе времени выбор времени исчезает, что позволяет перемещать вновь созданную точку останова.
  • Теперь при перетаскивании точек останова или сегментов автоматизации они привязываются к существующим временам точки останова. Модификатор «Shift» больше не требуется для перемещения сегментов автоматизации за существующие точки останова. Когда сетка отключена, точки останова автоматизации теперь привязаны к времени точки останова и времени сетки, если они находятся на расстоянии одного пикселя.
  • Для точек останова, захваченных мышью, значения автоматизации теперь отображаются при наведении курсора на точку останова или перетаскивании любого из следующего: сегмент автоматизации, сегмент автоматизации выбора времени или несколько точек останова в выделенном времени.
  • Теперь можно перемещать сегменты автоматизации по горизонтали.
  • Когда точки останова создаются близко к линии сетки, теперь они автоматически привязываются к сетке.
  • Вертикальная вспомогательная линия редактирования теперь появляется при перемещении точек останова в представлении компоновки или подробном представлении.
  • Точки останова теперь квантуются при перемещении.
  • В аранжировке введены ручки краев перехода, которые позволяют пользователям изменять длительность нарастания (с помощью «ручки начала плавного появления») и исчезновения (с помощью «конечной ручки затухания»), не влияя на пики затухания. Однако кромка затухания не может выходить за пределы пика затухания. Для этого необходимо использовать начало / конец клипа для увеличения размера этого клипа.
  • Удерживание «ALT» при изменении размера одной дорожки теперь изменяет размер всех дорожек в текущем просмотре.
    Браузер
  • Теперь можно отображать и искать неустановленные пакеты в браузере и устанавливать их оттуда.
  • На боковой панели «Пакеты» теперь отображаются обновления, внесенные в пакеты. Теперь пользователи могут загружать и устанавливать эти обновления.
  • После перезапуска Live ранее загруженные пакеты теперь готовы к установке, а частично загруженные пакеты теперь могут возобновить загрузку (если только Live не был перезапущен при измеренном интернет-соединении).
  • В браузере Live появилась функция «Цвета», которая позволяет помечать элементы браузера семью цветами на выбор. Цвета можно назначать элементам через пункты контекстного меню или с помощью цифровых клавиш (1-7 для назначения цветов и 0 для сброса). Элементу можно назначить несколько цветов, но будет показано не более трех цветов. Новые метки показывают все элементы, помеченные соответствующим цветом. Эти ярлыки можно переименовать через контекстное меню, а также отобразить / скрыть с помощью кнопки «Изменить» рядом с заголовком «Коллекции». При первом использовании отображается только метка красного цвета (с названием «Избранное»).
  • Улучшено поведение при использовании кнопки «Найти» для поиска пропавшей папки пользователя. Раньше это приводило к полному повторному сканированию новой папки, даже если ее содержимое было точно таким же, как и раньше, а также к потере всех цветов, назначенных элементам в папке.
  • При предварительном просмотре сэмплов в браузере сэмплы менее 2,5 секунд всегда будут воспроизводиться без искажений и без квантования. Сэмплы продолжительностью более 2,5 секунд не изменяются и будут воспроизводиться с квантованием, если не будет включена кнопка «Raw».
  • Иерархия групповых треков в Live Set теперь отображается в браузере. Это позволяет перетаскивать содержимое группового трека в текущий Live Set.
  • На боковой панели браузера теперь можно переименовывать ярлыки папок пользователя. Папки «Пользовательская библиотека» и «Текущий проект» теперь имеют контекстные меню с командами «Найти папку» / «Показать в Finder». Файлы можно перетаскивать в пользовательскую библиотеку, текущий проект и напрямую использовать метки папок.
    Захват
  • Представлен «Capture»: новый способ перенести музыку в Live без напряжения записи. Live теперь всегда прослушивает входные MIDI-данные на отслеживаемых треках (т. Е. Включенных треках с мониторингом, установленным на «Auto», или невооруженных треках с мониторингом, установленным на «In»). Нажатие кнопки «Захват» создает клип сеанса, содержащий последнюю проигранную фразу, без необходимости предварительно активировать запись или воспроизведение. Live немедленно воспроизводит захваченный цикл, чтобы вы могли добавить больше материала.
  • Чтобы запустить захват из нажатия 1 и 2, нажмите и удерживайте кнопку «Запись», а затем нажмите «Создать».
    Обработка файлов
  • В диалоговом окне «Экспорт аудио / видео» теперь отображаются переключатели «Экспорт PCM» и «Экспорт MP3». Когда "Экспорт PCM" включен, записывается файл PCM. Помимо WAV и AIFF для экспорта в PCM доступны форматы FLAC и WavPack. Когда "Экспорт MP3" включен, записывается файл MP3 CBR 320 кбит / с. Можно одновременно экспортировать PCM и MP3. Если ни один из переключателей не включен, кнопка «Экспорт» будет отключена.
  • После сохранения набора с помощью «Сохранить текущий набор», «Сохранить текущий набор как» или «Сохранить копию» предыдущая версия перемещается в резервную папку текущего проекта. В этой папке хранится до 10 старых версий Комплекта. В браузере Live текущий набор имеет команду контекстного меню «Показать старую версию», которая указывает на его самую последнюю версию резервной копии. Когда загружается резервная версия, она сохраняется в исходном месте.
  • При сохранении Live Set история отмены больше не очищается. Исходный файл и положение в истории запоминаются при создании, загрузке и сохранении Live-набора, а информация используется для восстановления Live-набора после сбоя.
  • Улучшено именование и сортировка записанных аудиофайлов за счет добавления отметки времени к имени файла. 
  • Добавлены различные изменения для Windows 10:
      - Live теперь поддерживает список переходов.
      - При параллельном использовании Live 9 и 10 они больше не запрашивают права администратора, если вы запускаете их попеременно.
      - Владельцы Live 9 и 10 могут выбрать, с какой версией Live открыть Live Set, щелкнув правой кнопкой мыши этот Set и выбрав «Открыть с помощью».
      - Теперь можно связать файлы .als с определенной Live-версией через «Панель управления» → «Приложения» → «Приложения по умолчанию».
  • Большие живые наборы теперь можно закрывать в 5-10 раз быстрее.
  • Повышена скорость загрузки образцов, на которые ссылается Live Set. Это значительно сокращает время загрузки Live Set.
  • Прекращена поддержка The Bridge.
    Улучшения интерфейса
  • Добавлен новый шрифт Ableton Sans в Live.
  • Представлена ​​цветовая палитра Live 10.
  • MIDI-ноты теперь отображаются в их цвете клипа в редакторе нот.
  • Обновлен внешний вид клипов как в режиме просмотра сеанса, так и в режиме аранжировки.
  • Введен набор из пяти новых тем вместо ранее доступных Live «скинов».
  • Введено предпочтение для включения режима HiDPI в Windows. Режим HiDPI можно включить в Windows 10 с помощью Live's Preferences → Look / Feel → Enable HiDPI mode.
  • Введен режим графического планшета и добавлен в настройки Live. Режим графического планшета позволяет использовать графические планшеты для значимого управления пользовательским интерфейсом Live и улучшает работу с сенсорными экранами. Он заменяет параметр «AbsoluteMouseMode».
  • Добавлены новые курсоры мыши для изменения размера клипов и перетаскивания содержимого клипа в Компоновке в Windows.
  • Циферблаты и ползунки Live теперь визуализируются с использованием векторной графики в реальном времени, а не растровых изображений. Результат выглядит резким и стабильным при любом уровне масштабирования.
  • В контекстное меню заголовков треков добавлен пункт меню «Назначить цвет трека клипам». Когда команда используется в представлении аранжировки, она применяет цвет дорожки к клипам представления аранжировки. При использовании в представлении сеанса цвет применяется к клипам представления сеанса.
  • Включены точки образца на осциллограммах. При увеличении масштаба пользователи теперь могут более внимательно видеть, где находятся их образцы на линейке времени.
  • Некоторые из движений пользовательского интерфейса Live были улучшены, чтобы двигаться со скоростью 60 кадров в секунду.
  • Добавлено несколько визуальных улучшений для Windows: во втором окне теперь отображается та же строка меню, что и в главном окне. При нажатии клавиши Alt буквы в строке меню больше не подчеркиваются. Кроме того, модальные окна теперь выводятся на передний план, если щелкнуть окно внизу. Раньше модальное окно можно было вывести на передний план только с помощью Alt + Tabbing в окно Live.
  • Ранее изменение размера представления сеанса по вертикали или горизонтали приводило к «подпрыгиванию» представления сеанса вверх и вниз, как способ согласования вертикального / горизонтального квантования сцен и клипов с измененным размером. Теперь размер окна сеанса изменяется плавно, без отскока или квантования.
  • Область перетаскивания клипа / устройства теперь всегда отображается в просмотре сеанса, чтобы можно было перетаскивать.
    Max для Live
  • Max теперь включен в Live и больше не требует внешней установки.
  • Live всегда будет использовать прилагаемую версию Max.
  • Max теперь загружается при запуске, а не при использовании первого устройства Max.
  • Устройства Max for Live теперь поддерживают несколько аудиовходов и выходов, к которым можно получить доступ через переключатели входного и выходного каналов трека. Устройства также можно направлять на произвольные треки через Live API.
  • Теперь можно направлять устройства и цепочки устройств на устройства Max for Live.
  • Микшер
  • Теперь можно изменить элемент управления панорамированием дорожки на «Режим разделения стерео панорамирования» через пункт контекстного меню. И то, и другое применимо как к расположению, так и к просмотру сеанса.
  • Теперь можно направить аудиовыход отдельного пэда Drum Rack на одну из цепей возврата его родительской Drum Rack. Это позволяет использовать эти возвратные цепи в качестве шин микширования.
  • Теперь можно сбросить значения регулятора и ползунка до значений по умолчанию, дважды щелкнув их (так же, как работает сочетание клавиш «Удалить»). Жест двойного щелчка не применяется к переключателям или кнопкам действий.
    Редактирование мульти-MIDI-клипов
  • Введено редактирование нескольких клипов, которое позволяет просматривать до семи клипов MIDI одновременно. Когда выбрано несколько MIDI-клипов, щелчок по нотной полосе или полосе цикла нескольких клипов в MIDI-редакторе устанавливает фокус на соответствующий клип. Точки петли представлены вертикальными линиями в редакторе заметок.
  • Состояние кнопки Fold редактора MIDI теперь устанавливается глобально для всех треков, а не для трека. Кнопка Fold теперь может быть привязана к клавишам / MIDI.
  • Состояние переключателя предварительного просмотра редактора MIDI теперь устанавливается глобально для всех дорожек, а не для каждой дорожки.
  • Изменено поведение кнопки Fold для MIDI-клипов на дорожках, которые содержат Drum Rack: когда Fold деактивирован, редактор MIDI Note отображает только строки с нотами, соответствующими пэду с устройствами на нем, а когда Fold активирован, отображаются только строки, содержащие ноты.
    Навигация
  • «Добавлен« Показать все треки »в меню просмотра и контекстное меню в режиме аранжировки. Это сводит к минимуму все дорожки, позволяя видеть как можно больше на экране. Действие также можно запустить с помощью ярлыка «S».
  • Теперь можно увеличивать масштаб (и обратно) к выбранному времени в расположении и в подробном представлении. Эта функция может быть активирована с помощью «Z» и Shift + «Z» соответственно, через меню «Вид» или через пункт контекстного меню.
  • Теперь можно масштабировать по вертикали и горизонтали в представлении компоновки, используя жесты трекпада двумя пальцами или колесо мыши. Нажатие «ALT» (Mac / Win) при прокрутке по вертикали увеличивает масштаб оси амплитуды / высоты тона выбранной дорожки (дорожек). Нажатие «CMD» (Mac) / «CTRL» (Win) во время вертикальной прокрутки увеличивает масштаб временной шкалы до положения курсора.
  • Теперь можно выполнять горизонтальную прокрутку с помощью клавиши-модификатора Shift также в WIndows.
  • На американских клавиатурах теперь можно увеличивать масштаб, нажимая «+» без использования клавиши-модификатора Shift.
  • В режимах «Компоновка», «Подробный вид», «Упрощенный» и «Сэмплер» масштабирование с помощью колеса мыши или трекпада теперь возможно с помощью клавиши-модификатора Ctrl (WIN) или Cmd (OSX).
  • Когда время выбрано в клипе аранжировки, отображение подробного просмотра теперь приближается к выбранному времени.
  • При редактировании в представлении «Расположение» или «Подробный обзор» функция «Follow» теперь приостанавливается, а не деактивируется. Во время паузы цвет кнопки Follow меняет цвет с желтого на оранжевый. При остановке или повторном запуске воспроизведения или при прокрутке функция Follow запускается снова. Кроме того, Follow также приостанавливается при горизонтальной прокрутке в Arrangement.
  • Перетаскивание закрепленных клипов и хвостов в область перетаскивания клипа / устройства в представлении аранжировки создает новые замороженные дорожки.
  • При перетаскивании клипов из существующих дорожек в область перетаскивания клипов / устройств создается новая дорожка с этими клипами, а также с устройствами из исходных дорожек.
  • Звуковые эффекты теперь можно перетащить в область запуска сцены мастер-трека.
    Настройки
  • Теперь можно задать имена для моно- или стереофонических входных и выходных каналов из окна предпочтений «Конфигурация входа / выхода». Эти имена появляются во всех раскрывающихся списках выбора каналов ввода / вывода (Примечание: имена всегда связаны только с конкретным аудиоустройством; они не передаются никаким другим устройствам). Команды клавиш «Tab» и «Shift-Tab» могут использоваться для переключения между названиями каналов.
  • Теперь можно выбирать между различными звуками метронома из контекстного меню метронома.
  • Теперь можно выбрать другой интервал тактов для метронома с помощью его раскрывающихся и контекстных меню. Значение по умолчанию «Авто» такое же, как и в Live 9 (т. Е. Тиковый интервал следует за знаменателем размера). Выбор интервалов, которые не помещаются в одну полосу текущего размера, будет отключен. Если текущий выбранный интервал больше не помещается в полосе из-за изменения тактового размера, тик метронома вернется в режим «Авто». Однако, если размер изменен таким образом, что выбранный интервал снова помещается в полосу, отметка будет следовать за выбранным интервалом.
  • В раскрывающееся и контекстное меню метронома добавлен пункт меню «Включить только во время записи». Если этот параметр включен и активирован метроном, он слышен только во время записи. Если выполняется запись в аранжировку при активном переключателе Punch-In, метроном слышен только после точки вставки. Параметр «Включить только во время записи» не сохраняется в настройках и должен включаться каждый раз при запуске Live.
  • Компьютерная MIDI-клавиатура теперь всегда выключена при запуске приложения. Теперь его можно переключать с помощью клавиши «M».
  • По умолчанию компьютерная MIDI-клавиатура теперь всегда отключена при запуске. Его текущее состояние сохраняется при создании или загрузке Live Set.
  • Производительность 64-разрядной версии Live в Windows теперь должна быть немного улучшена.
  • В Windows маленькая стрелка для выхода из полноэкранного режима теперь также видна в правом нижнем углу второго окна.
  • В Windows авторизация Live после переустановки операционной системы не требует использования другой авторизации.
  • Обновлен FLAC до 1.3.2 и включена оптимизация кода. Live Packs теперь может устанавливаться на 20% быстрее, чем раньше на современных процессорах.
  • Пункты контекстного меню теперь реагируют на отпускание мыши, а не после первого щелчка. Это позволяет перетащить выделение, чтобы отменить запуск элемента, при этом контекстное меню остается открытым. При наведении курсора на отключенные элементы контекстного меню они больше не выделяются.
